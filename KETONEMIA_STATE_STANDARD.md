# Ketonemia State Standard

The **Ketonemia State Standard (KSS)** organizes blood ketone signal language into a governed reference model.

KSS does not invent medical thresholds. It organizes cited guidance and preserves the clinical boundary.

## Initial State Model

| Code | State | Reference Meaning |
| --- | --- | --- |
| K0 | Baseline | Low or non-prominent ketone signal. |
| K1 | Low Presence | Detectable ketone presence requiring context before meaning is assigned. |
| K2 | Nutritional / Fasting Range | Signal that may appear in fasting or nutritional contexts. |
| K3 | Elevated Context Signal | Reading that requires context and should not be interpreted as a standalone verdict. |
| K4 | Clinical Concern Boundary | Concern zone where cited guidance may recommend professional medical contact or assessment. |
| K5 | Emergency Referral Boundary | Urgent-care language zone when cited guidance supports emergency referral. |

## Threshold Rule

KSS must never publish internally invented clinical cutoffs.

When numeric ranges are used:

- cite the external source;
- identify the population or context;
- state the measurement type;
- avoid universal claims;
- distinguish educational framing from medical advice.

## Allowed Use

KSS may be used for:

- reference pages;
- signal map outputs;
- AI reference packs;
- educational diagrams;
- licensed terminology;
- source-indexed summaries.

## Prohibited Use

KSS must not be used to:

- diagnose DKA;
- declare a reader safe;
- recommend treatment;
- replace clinician judgment;
- automate emergency triage;
- create unsourced thresholds.
